# Contributing to ProvChain

Thank you for your interest in contributing to ProvChain! We welcome contributions from the community to help improve this project.

## Code of Conduct

Please adhere to professional conduct. Be respectful and constructive in your communications.

## How to Contribute

### Reporting Bugs

1.  Check the [Issues](https://github.com/Sanjog-Poojary/DocReview/issues) tab to see if the bug has already been reported.
2.  If not, open a new issue with a clear title and description.
3.  Include steps to reproduce, expected behavior, and screenshots if applicable.

### Suggesting Enhancements

1.  Open a new issue describing the feature or improvement.
2.  Explain why it would be beneficial to the project.

### Pull Requests

1.  **Fork** the repository.
2.  Create a new branch for your feature or fix: `git checkout -b feature/amazing-feature`.
3.  Commit your changes: `git commit -m 'feat: add amazing feature'`.
4.  Push to the branch: `git push origin feature/amazing-feature`.
5.  Open a **Pull Request**.

## Development Workflow

1.  Ensure you have Node.js and Git installed.
2.  Follow the setup instructions in [README.md](./README.md).
3.  Run tests before submitting:
    - Contracts: `cd contracts && npm test`
    - Frontend: `cd frontend && npm run lint`

## Style Guide

-   **Frontend**: Follow existing Next.js and TailwindCSS patterns.
-   **Contracts**: Adhere to Solidity style guides and security best practices (OpenZeppelin).
